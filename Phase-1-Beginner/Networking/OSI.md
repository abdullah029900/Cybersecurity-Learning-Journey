# 📡 OSI Model (Open Systems Interconnection Model)

## 📘 Overview

The OSI Model is a **7-layer framework** used to understand how data travels from one device to another over a network. Each layer has a specific role in ensuring smooth communication.

---

## 🧩 7 Layers of OSI Model

### 1️⃣ Physical Layer

* Deals with **hardware and signal transmission**
* Sends raw bits (0s and 1s)
* Mediums: Ethernet cables, Wi-Fi signals

📌 Example: Data is converted into electrical or radio signals and transmitted.

---

### 2️⃣ Data Link Layer

* Ensures **error-free communication between devices**
* Uses **MAC addresses**
* Data is packaged into **frames**

📌 Example: Laptop communicates with router using MAC address.

---

### 3️⃣ Network Layer

* Responsible for **routing and path selection**
* Uses **IP addresses**

📌 Example: Data finds the best route from one network to another.

---

### 4️⃣ Transport Layer

* Ensures **reliable data transfer**
* Breaks data into segments
* Protocols: TCP / UDP

📌 Example: Ensures all parts of a file arrive correctly.

---

### 5️⃣ Session Layer

* Manages **sessions (connections)**
* Starts, maintains, and ends communication

📌 Example: Keeps your session active while browsing a website.

---

### 6️⃣ Presentation Layer

* Handles **data formatting, encryption, and compression**

📌 Example: Encrypts data for secure communication (HTTPS).

---

### 7️⃣ Application Layer

* Closest to the user
* Provides network services to applications

📌 Example: Web browsers, email clients.

---

## 🔄 Data Flow (Simple Explanation)

1. Data starts from Application Layer
2. Moves down to Physical Layer (converted into signals)
3. Travels through network
4. At receiver side, it moves back up to Application Layer

## 🎯 Key Takeaways

* OSI Model standardizes network communication
* Each layer has a specific function
* Helps in troubleshooting and understanding networks
